  <decision>Choose storage backend: Cloudflare R2 or Supabase Storage</decision>
  <context>
Both are free tier options. R2 provides 10GB free storage, Supabase Storage provides 1GB on free tier. R2 requires separate service setup, Supabase Storage integrates with existing Supabase project from Plan 01.
  </context>
  <options>
    <option id="r2">
      <name>Cloudflare R2</name>
      <pros>
- More generous free tier (10GB vs 1GB)
- S3-compatible API (standard, well-documented)
- Better for large file volumes
- Independent from Supabase (decoupled)
      </pros>
      <cons>
- Requires additional service setup (Cloudflare account)
- More environment variables to manage
- Extra API credentials to secure
      </cons>
    </option>
    <option id="supabase-storage">
      <name>Supabase Storage</name>
      <pros>
- Integrated with existing Supabase project
- Reuses existing credentials (no new account needed)
- Simpler mental model (single dashboard)
- Built-in integration with Supabase auth/database
      </pros>
      <cons>
- Smaller free tier (1GB)
- Less generous for scaling
- Couples storage with database (less flexible)
      </cons>
    </option>
  </options>

  <instructions>
**If you selected R2:**

1. Visit https://dash.cloudflare.com
2. Sign in or create account (free tier)
3. Navigate to R2 Object Storage in sidebar
4. Click "Create bucket"
   - Name: type-it-up-uploads
   - Location: Automatic (or nearest region)
5. Go to "Manage R2 API Tokens"
6. Click "Create API Token"
   - Token name: type-it-up-api
   - Permissions: Object Read & Write
   - Bucket scope: type-it-up-uploads
7. Copy Account ID, Access Key ID, Secret Access Key
8. Add to .env.local:

```
R2_ACCOUNT_ID=your_account_id
R2_ACCESS_KEY_ID=your_access_key_id
R2_SECRET_ACCESS_KEY=your_secret_access_key
R2_BUCKET_NAME=type-it-up-uploads
```

**If you selected Supabase Storage:**

1. Go to Supabase Dashboard -> Storage
2. Click "Create bucket"
   - Name: uploads
   - Public bucket: NO (we'll use signed URLs)
3. No additional env vars needed (reuses Plan 01 credentials)
